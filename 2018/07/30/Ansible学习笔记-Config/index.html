发表于 2018-07-30 | 分类于 Ansible

Ansible配置文件笔记

Ansible读取配置文件的顺序

  • ANSIBLE_CONFIG（如果设置了环境变量）

  • ansible.cfg（当前目录）

  • ~/.ansible.cfg（当前用户宿主目录）
  • /etc/ansible.cfg

命令

语法

options

--version — 版本号

-c <CONFIG_FILE>, --config <CONFIG_FILE> — 指定配置文件路径

-v, --verbose — 详细模式（-vvv表示更多，-vvvv表示开启连接调试）

view

显示当前的配置文件

dump

显示当前设置，如果指定了配置文件，则覆盖配置文件

--only-changed — 仅显示从默认值更改的配置

list

列出lib/constants.py的所有配置，并显示env和配置文件的名称

配置

在此记录一下配置文件各个选项的作用：

以下配置项在实际文件中都被注释，大多数情况下使用默认配置，根据自己的需要修改。

为了显示效果，我把注释都去掉了。

defaults

日常可能用到的配置，多数保持默认即可。

inventory

主机清单的相关配置。

privilege_escalation

sudo用户提权的相关配置。

paramiko_connection

paramiko连接相关配置，目前已经不常用，基本适用ssh连接。

ssh_connection

ssh连接相关配置。

persistent_connection

持久化连接的相关配置。

accelerate

accelerate模式（加速模式）的相关配置。

selinux

selinux的相关配置。

colors

颜色的相关配置。

diff

打印差异的相关配置。
